## Root cause closed by R1

M23.7.5 used:

```text
query_text = sample["section_id"]
```

That compared identifier-string embeddings rather than the semantic intent represented
by the held-out evaluation suite. R1 removes this defect from the future observation
contract by requiring humanised semantic topics and class-specific synthetic templates.

## Accepted probe semantics

The versioned manifest contains eight slots using only positive, ACL-allowed cases:

- two `direct-fact` probes;
- two `terminology` probes;
- two `cross-section` probes;
- two `provenance` probes.

At future R3 runtime, exactly eight public non-production samples must be sorted by point
ID and bound to those slots. The compiler humanises `concept_id`, `section_id`,
`article_id`, `document_id` and `source_path`, removes generic or opaque tokens, requires
at least three semantic tokens, and applies the query-class template.

Each compiled probe keeps the offline case identity and binds the exact live target
section as its expected relevance set.

## Privacy proof

Compiled raw query text exists only in memory. Durable report mappings contain only:

- probe and offline case identity;
- query class;
- point and target section identity;
- expected relevant section IDs;
- query digest;
- token and character counts;
- explicit false raw-query and raw-answer persistence flags.

No user query, answer text, credential, service URL or arbitrary exception text is
persisted.
