# M26.8 Synthetic Preview Evidence Integration and Candidate Bundle

M26.8 consumes the accepted M26.7 non-serving preview packages and compiles deterministic synthetic candidate bundles for internal contract validation.

## Authority boundary

M26.8 is still synthetic-only. It does not call a live provider, use credentials, bind a real corpus, enable semantic or hybrid production serving, serve an answer, change a production pointer, mutate R2 or Qdrant, or create verified final answers.

## Candidate bundle contract

The stage records only identity and evidence structure:

- presentation package SHA-256;
- evaluation and draft package SHA-256 identities;
- candidate claim IDs;
- candidate citation/binding IDs;
- warning banners;
- refusal reason codes;
- bundle-level candidate record digests.

Answer text remains empty. Displayable claim content remains absent. Refusal previews remain fail-closed records with no claim or binding identities.

## Acceptance discipline

Implementation evidence is generated by an exact-head workflow. M26.9 remains blocked until a separate reconciliation PR records `m26_8_preview_evidence_candidate_bundle_accepted`.
